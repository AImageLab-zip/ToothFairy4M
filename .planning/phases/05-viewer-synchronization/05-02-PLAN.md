Phase 5 synchronized scrolling implementation with:

  <how-to-verify>
**Prerequisites:**
1. Start Django dev server: `python manage.py runserver`
2. Navigate to a brain patient detail page with at least 2 available modalities (T1, T2, FLAIR, or T1c)
3. Open browser console to see synchronization debug logs

**Test Case 1: Default synchronization (SYNC-01, SYNC-02)**
1. Drag T1 modality to top-left window (Window 0)
2. Drag T2 modality to top-right window (Window 1)
3. Both windows should be in axial orientation (A button active)
4. Scroll in the T1 window using mouse wheel
5. **Expected:** T2 window scrolls to the same slice automatically
6. Scroll in the T2 window
7. **Expected:** T1 window follows
8. Check console for "sliceIndexChanged" events

**Test Case 2: Different orientations operate independently (SYNC-05)**
1. Keep T1 in axial (top-left)
2. Click "S" button in T2 window (top-right) to switch to sagittal
3. Scroll in the T1 window (axial)
4. **Expected:** T2 window (sagittal) does NOT scroll
5. Scroll in the T2 window (sagittal)
6. **Expected:** T1 window (axial) does NOT scroll

**Test Case 3: Same orientation after change (SYNC-01)**
1. Drag FLAIR to bottom-left window (Window 2)
2. Window 2 starts in axial orientation
3. Scroll in T1 window (top-left, axial)
4. **Expected:** Both T2 (if switched back to axial) and FLAIR scroll together
5. Click "S" button in FLAIR window to switch to sagittal
6. Click "S" button in T1 window to switch to sagittal
7. Scroll in T1 window
8. **Expected:** FLAIR window (now both sagittal) scrolls with T1

**Test Case 4: Free Scroll toggle (SYNC-03, SYNC-04)**
1. Set T1 (top-left) and T2 (top-right) both to axial orientation
2. Click the link icon (<i class="fas fa-link"></i>) in the T2 window's top-right corner
3. **Expected:** Button icon changes to <i class="fas fa-link-slash"></i> and button becomes yellow
4. Scroll in T1 window
5. **Expected:** T2 window does NOT scroll (free-scroll mode active)
6. Scroll T2 window to a different slice manually
7. Click the link-slash icon again in T2 window
8. **Expected:** T2 window immediately jumps to the same slice as T1 (re-synced)
9. **Expected:** Button icon changes back to <i class="fas fa-link"></i> and normal color
10. Scroll in T1 window
11. **Expected:** T2 window scrolls together again (synchronized)

**Test Case 5: Four windows with mixed orientations**
1. Load T1, T2, FLAIR, T1c in all four windows
2. Set top row (Windows 0, 1) to axial
3. Set bottom row (Windows 2, 3) to sagittal
4. Scroll in Window 0 (T1, axial)
5. **Expected:** Window 1 scrolls (both axial), Windows 2 and 3 do NOT scroll (sagittal)
6. Scroll in Window 2 (FLAIR, sagittal)
7. **Expected:** Window 3 scrolls (both sagittal), Windows 0 and 1 do NOT scroll (axial)

**Test Case 6: Clear and reload**
1. Right-click on Window 1, select "Clear"
2. Scroll in Window 0
3. **Expected:** Only remaining axial windows scroll, cleared window ignored
4. Drag T2 back into Window 1
5. **Expected:** Window 1 re-joins axial sync group automatically
6. Scroll in Window 0
7. **Expected:** Window 1 scrolls together again

**Console checks:**
- Look for "Window X: orientation → orientation" logs showing group updates
- Look for "Groups:" logs showing synchronizationGroups state
- Should NOT see any JavaScript errors or warnings
- onLocationChange callbacks should fire when scrolling

**Help modal verification:**
1. Click help icon in brain patient detail page
2. Scroll to "Navigation Controls" section
3. **Expected:** Documentation for Free Scroll feature with link icons

**Pass criteria:**
✓ Same-orientation windows scroll together by default
✓ Different-orientation windows operate independently
✓ Free Scroll button breaks synchronization (icon changes, yellow highlight)
✓ Re-sync button rejoins group at current slice
✓ Orientation changes update sync group membership correctly
✓ Console shows synchronization events with no errors
✓ Help modal documents Free Scroll feature
