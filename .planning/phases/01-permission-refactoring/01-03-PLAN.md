  <files>maxillo/views/auth.py</files>
  <action>
Update the auth view to use ProjectAccess instead of creating MaxilloUserProfile or BrainUserProfile.

**Current code pattern:**
```python
from maxillo.models import MaxilloUserProfile
from brain.models import BrainUserProfile

# On user registration:
profile, created = MaxilloUserProfile.objects.get_or_create(
    user=user,
    defaults={'role': 'standard'}
)
# or
profile, created = BrainUserProfile.objects.get_or_create(
    user=user,
    defaults={'role': 'standard'}
)
```

**Replace with:**
```python
from common.models import Project, ProjectAccess

# On user registration, determine which project based on context
# (likely based on invitation or session)
project_slug = request.session.get('registration_project', 'maxillo')
try:
    project = Project.objects.get(slug__iexact=project_slug)
    access, created = ProjectAccess.objects.get_or_create(
        user=user,
        project=project,
        defaults={'role': 'standard'}
    )
except Project.DoesNotExist:
    # Fallback to maxillo project or handle error
    pass
```

**Remove imports:**
```python
# DELETE these lines:

from brain.models import BrainUserProfile
```

**Add imports:**
```python
from common.models import Project, ProjectAccess
```

- This may be based on an invitation link, session data, or URL path
- Review the existing registration logic to understand the project context
- If an invitation is accepted, the ProjectAccess should be created with the invitation's role

**1. Update maxillo/views/profile.py:**

The profile view needs to pass the target user's ProjectAccess to the template.

Find the profile view function (likely named `user_profile` or similar) and update it to:

```python
from common.models import Project, ProjectAccess

def user_profile(request, username=None):
    # ... existing code to get target_user ...

    # Get target user's ProjectAccess for current project
    current_project_id = request.session.get('current_project_id')
    target_profile = None
    if current_project_id:
        try:
            target_profile = ProjectAccess.objects.get(
                user=target_user,
                project_id=current_project_id
            )
        except ProjectAccess.DoesNotExist:
            target_profile = None

    context = {
        # ... existing context ...
        'target_profile': target_profile,
    }

    return render(request, 'maxillo/user_profile.html', context)
```

**Key change:** Explicitly look up ProjectAccess for the target user (not the request user). The target user doesn't have `user.profile` set by middleware because only `request.user` gets that.

**2. Update templates/maxillo/user_profile.html:**

Find references to `target_user.profile` and replace with `target_profile`:

```html
<!-- OLD: -->
<span class="badge bg-secondary">{{ target_user.profile.get_role_display }}</span>

<!-- NEW: -->
<span class="badge bg-secondary">{{ target_profile.get_role_display }}</span>
```

Also check for conditions like:
```html
<!-- OLD: -->
{% if user.profile.can_view_other_profiles and target_user.profile.is_admin %}

<!-- NEW: -->
{% if user.profile.can_view_other_profiles and target_profile.is_admin %}
```

**Note:** `user.profile` (the logged-in user) still works because middleware sets it. Only `target_user.profile` needs to be replaced with `target_profile` from context.
