```javascript
/**
 * MaxilloNiiVueViewer - Fixed 2x2 CBCT viewer using NiiVue
 *
 * Manages three NiiVue instances (Axial, Sagittal, Coronal) with:
 * - Synchronized crosshair positions across all views
 * - Percent-based windowing controls via adapter
 * - Integration with existing Maxillo template structure
 */

(function() {
    'use strict';

    function MaxilloNiiVueViewer() {
        this.initialized = false;
        this.loading = false;
        this.volumeBlob = null;
        this.targetModality = 'cbct';
        this.containerPrefix = '';

        // NiiVue instances for each orientation
        this.viewers = {
            axial: null,
            sagittal: null,
            coronal: null
        };

        // Windowing state (percent-based for UI compatibility)
        this.windowPercentMin = 0;
        this.windowPercentMax = 100;

        // Synchronization state
        this._isSyncing = false;
        this._syncRAF = null;
    }
```

Key methods to implement:

1. `init(modalitySlug)` - Entry point, matches CBCTViewer.init() signature
   - Skip for image modalities (intraoral, teleradiography, panoramic)
   - Fetch volume data from API
   - Call `initializeViewers()` on success

2. `loadVolumeData()` - Fetch NIfTI from API
   - Check preload cache first (window._volumePreloadCache)
   - URL: `/${projectNamespace}/api/patient/${scanId}/cbct/` for CBCT
   - URL: `/${projectNamespace}/api/patient/${scanId}/volume/${modality}/` for other volumes
   - Store as Blob for NiiVue

3. `initializeViewers()` - Create 3 NiiVue instances
   - Create canvas elements in each container (axialView, sagittalView, coronalView)
   - Instantiate NiiVueViewer for each orientation
   - Call `setupSynchronization()` after all viewers ready

4. `setupSynchronization()` - Cross-view sync
   - Attach `onSliceChange` callback to each viewer
   - On change: update other viewers' crosshairPos using drawScene()
   - Use rAF throttling to coalesce updates (match viewer_grid.js pattern)

5. `setWindowingFromPercent(percentMin, percentMax)` - Windowing adapter
   - Update all three viewers' windowing via setWindowing()
   - Store in this.windowPercentMin/Max for state tracking

6. `dispose()` - Cleanup
   - Dispose all NiiVue instances
   - Remove canvases from DOM
   - Reset state (but keep volumeBlob cached)

7. `refreshAllViews()` - Force redraw (for tab switching)

Expose as `window.MaxilloNiiVueViewer = MaxilloNiiVueViewer;`

Match CBCTViewer API surface so template JavaScript can use it as drop-in replacement.

       ```javascript
       this.viewers[orientation].onSliceChange((msg) => {
           if (this._isSyncing) return;
           this._pendingCrosshairPos = [...this.viewers[orientation].nv.scene.crosshairPos];
           this._pendingSourceOrientation = orientation;
           if (!this._syncRAF) {
               this._syncRAF = requestAnimationFrame(() => this._applyCrosshairSync());
           }
       });
       ```

2. Add `_applyCrosshairSync()` private method:
   ```javascript
   MaxilloNiiVueViewer.prototype._applyCrosshairSync = function() {
       this._syncRAF = null;
       this._isSyncing = true;

       const sourceOrientation = this._pendingSourceOrientation;
       const crosshairPos = this._pendingCrosshairPos;

       for (const orientation of ['axial', 'sagittal', 'coronal']) {
           if (orientation === sourceOrientation) continue;
           const viewer = this.viewers[orientation];
           if (viewer && viewer.isReady() && viewer.nv) {
               const pos = viewer.nv.scene.crosshairPos;
               pos[0] = crosshairPos[0];
               pos[1] = crosshairPos[1];
               pos[2] = crosshairPos[2];
               viewer.nv.drawScene();  // Fast GPU redraw, not updateGLVolume
           }
       }

       this._isSyncing = false;
   };
   ```

3. Add slice counter updates:
   - After sync, update slice counters for all views
   - Format: `${currentSlice + 1} / ${totalSlices}`
   - Use existing `.slice-counter` element pattern from cbct.js
